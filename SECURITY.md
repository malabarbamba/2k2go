# Security Policy

## Supported Versions

| Version | Supported |
| --- | --- |
| `main` (latest) | Yes |
| Older commits / stale forks | No |

## Reporting a Vulnerability

Do not open a public issue for security reports.

Use GitHub Private Vulnerability Reporting for this repository:

1. Open the repository **Security** tab.
2. Click **Report a vulnerability**.
3. Include impact, reproduction steps, affected paths, and suggested remediation.

If private reporting is unavailable, contact maintainers through private GitHub channels and clearly mark the message as a security report.

## Response Targets

- Acknowledge report within 72 hours.
- Initial triage within 7 days.
- Fix timeline depends on severity and complexity.
- Public disclosure happens after a fix is available (coordinated disclosure).

## Scope

In scope:

- Source code vulnerabilities.
- Build, CI/CD, and release workflow vulnerabilities.
- Credential leaks and sensitive data exposure in tracked files.
- Dependency and supply-chain risks that affect this repository.

Out of scope:

- Social engineering without a technical exploit path in this repository.
- Reports requiring physical access to maintainer hardware.
